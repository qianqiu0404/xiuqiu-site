# RWA 与美股上链：你持有的究竟是什么

“美股上链”听起来像是把一股股票铸造成一个 Token，然后在区块链上自由转账。

真实结构通常没有这么直接。

链上 Token 只是最终呈现给用户的资产接口。它背后可能还存在发行实体、SPV、托管机构、证券账户、销售条款、KYC、司法辖区限制和赎回流程。

所以研究 RWA，第一步不是看 Token 名称，而是回答：

```text
Token 对应什么法律权利
底层资产由谁持有
持有人如何申购和赎回
发行人失败时如何主张权利
链上转账受到哪些限制
```

本文以代币化美股为主要场景，并用 Ondo Global Markets 的公开结构作为案例。它不是所有 RWA 产品的统一模板，而是帮助理解“链上资产展示”和“链下权利结构”如何连接。

# 系统如何运作

一个常见的代币化证券结构可以抽象成：

```text
用户完成 KYC / 资格检查
-> 用户提交申购并支付稳定币或法币
-> 发行实体或其合作方购买底层证券
-> 证券由托管机构持有
-> 链上合约铸造对应 Token
-> Token 在允许的地址和网络中流通
-> 用户赎回时销毁 Token
-> 发行结构出售或交割底层资产
-> 用户收到稳定币、法币或约定资产
```

这里至少存在两套状态。

第一套是链下金融状态：证券是否买入、托管、分红、拆股、停牌或出售。

第二套是链上 Token 状态：铸造量、持有人、转账、冻结、销毁和跨链供应量。

一个生产级 RWA 系统必须让两套状态持续对齐。链上总供应量不能脱离托管资产，企业行动也不能只在链下发生而不反映到持有人权益。

# 股票、SPV 和 Token 不是一件东西

理解代币化美股时，需要把几个角色拆开。

## 底层股票

这是传统市场中的证券，例如某家上市公司的普通股。它存在于证券市场、经纪商和托管体系中。

## 发行实体或 SPV

SPV 是为特定发行或资产隔离目的设立的特殊目的实体。它可以持有或通过托管安排控制底层资产，并依据销售条款向 Token 持有人承担义务。

## 链上 Token

Token 是用户在钱包里看到、可以被合约识别的链上凭证。它代表的具体权利取决于法律文件和发行结构，而不是取决于它是不是 ERC20。

因此，下面两句话可能同时成立：

```text
Token 的价值跟踪某只股票
Token 持有人并不是该上市公司的登记股东
```

# Ondo 案例：结构化票据而不是直接持股

截至 2026 年 6 月，Ondo Global Markets 的官方法律说明把其代币化股票描述为由 BVI 发行实体发行的 structured note，也就是结构化票据。发行实体通过受监管的托管经纪商持有对应底层证券，并为 Token 持有人设置相应的担保权益。

官方文件同时说明，Token 持有人可以按相应条款获得底层资产价值变化带来的经济敞口和赎回价值，但不拥有底层公司的股东投票权、股东信息权等传统股东权利。

这意味着用户需要区分：

```text
价格和企业行动带来的经济敞口
!=
直接登记持股和完整股东权利
```

此外，发行和销售受到 KYC、司法辖区、制裁及证券法规限制。能在二级市场接收 Token，不等于一定拥有直接向发行方申购或赎回的资格。

这个案例的重要性不在于评价产品好坏，而在于展示一个事实：智能合约无法替代法律权利说明。

# 价值在哪里捕获

RWA 方向里至少有三类不同资产：

| 资产 | 可能拥有的权利 | 不应自动假设的权利 |
| --- | --- | --- |
| 代币化股票或票据 | 底层资产经济敞口、合同约定赎回权 | 直接股东权利 |
| 发行平台公司股权 | 公司利润和治理相关权利 | 底层资产本身的直接所有权 |
| 协议治理代币 | 治理、激励或特定协议用途 | 公司股权、全部平台收入 |

因此，“RWA 规模增长”并不能直接推出“某个 RWA 治理代币应该增值”。

价值捕获必须落到具体机制：发行费、管理费、赎回费、利差、协议收入、回购、质押收益或股权分配。没有明确连接时，品牌相关性不能替代现金流权利。

# 参与者承担什么风险

代币化证券把传统金融风险和链上风险叠在了一起。

## 发行人与法律结构风险

Token 持有人的请求权通常指向发行实体，而不是直接指向上市公司。需要理解破产隔离、担保权益、适用法律和争议处理机制。

## 托管与经纪风险

底层资产需要由经纪商或托管方持有。链上合约运行正常，不代表链下证券一定可用、未被冻结或能够及时出售。

## 稳定币与结算风险

申购、交易和赎回经常以稳定币结算。用户同时承担稳定币发行人、链和结算通道风险。

## 合约与跨链风险

Token 可能支持多链流通。每增加一条链，就增加合约升级、桥接、权限和跨链供应量对账的复杂度。

## 流动性与折溢价风险

二级市场价格可能偏离对应证券净值。传统证券休市而链上市场持续交易时，价格发现和套利能力尤其容易发生变化。

## 合规与冻结风险

受监管资产通常不会拥有完全无许可的转账能力。合约可能包含地址资格、暂停、冻结或强制转移机制。司法辖区变化也可能影响申购、持有和赎回。

# 钱包接入 RWA 资产时要做什么

普通 ERC20 接入常常只需要合约地址、symbol、decimals 和余额查询。RWA 资产远远不够。

钱包后端至少需要额外维护：

```text
发行实体和法律文件
底层资产标识和交易时段
支持的网络与合约地址
地址是否满足持有和转账资格
申购、赎回和暂停状态
净值、市场价格和更新时间
企业行动与分配记录
冻结、黑名单和司法辖区提示
```

交易前不能只执行余额检查，还可能需要检查发送方和接收方资格：

```text
用户发起 RWA Token 转账
-> 钱包读取资产限制
-> 检查目标地址资格
-> 模拟合约调用
-> 展示可能的冻结或失败原因
-> 用户签名
-> 广播并跟踪链上状态
```

如果钱包提供申购和赎回入口，还要把链下订单状态与链上交易关联起来：

```text
submitted
-> compliance_checking
-> payment_confirmed
-> asset_purchasing
-> token_minted
-> completed
```

这里的状态机和交易所钱包充值提现很相似：链上成功只是一个阶段，业务完成还依赖发行方确认、资产交割和账务记录。

# 作为建设者可以做什么

RWA 的建设机会集中在链上与链下边界，而不是重复发行一个资产名称相似的 Token。

可以落地的方向包括：

- 展示发行实体、托管方、底层资产和赎回权的数据终端；
- 统一多发行方的净值、交易时段、企业行动和资产状态；
- 为钱包提供受限 Token 的转账预检和失败原因解释；
- 对账托管资产、链上供应量和多链流通量；
- 为用户生成申购、赎回、分红和税务记录；
- 在借贷场景中识别 RWA 的流动性、冻结和预言机风险。

这些工具真正服务的是“可信地知道自己持有什么”，而不仅是让资产在钱包里显示一个图标。

# 最后理解

RWA 的核心不是把现实资产名字写进区块链，而是把链下权利、托管资产和链上状态连接起来。

分析代币化美股时，我会沿着这条链路逐层确认：

```text
底层证券
-> 托管机构
-> 发行实体 / SPV
-> 合同权利
-> 链上 Token
-> 持有人资格
-> 申购与赎回
```

只有这条链路完整，链上 Token 才不只是一个跟随股票名字波动的符号。
